## Overview

The Flourishing project integrates **four major data sources** for f-factor analysis:

| Source | Level | Coverage | Access | Script |
|--------|-------|----------|--------|--------|
| **World Happiness Report 2024** | Country | ~137 countries | GitHub mirror (free) | `01_download_wellbeing_data.py` |
| **OECD Better Life Index** | Country | 38 OECD countries | GitHub mirror (free) | `01_download_wellbeing_data.py` |
| **World Bank Open Data** | Country | 200+ countries/territories | REST API (free) | `01_download_wellbeing_data.py` |
| **MIDUS 2** | Individual | ~4,963 US adults | ICPSR (registration required) | `04_individual_level_f_analysis.py` |

<Info>
**Two analysis levels**:
- **Ecological (country-level)**: WHR + OECD + World Bank → `wellbeing_merged.csv`
- **Individual-level**: MIDUS 2 → `midus_scored.csv`

Ecological correlations are _inflated_ due to aggregation. Both levels are needed to validate the mutualism model.
</Info>

## World Happiness Report 2024 (WHR)

### Description

The **primary data source** for country-level PERMA+H analysis. WHR aggregates Gallup World Poll data into national averages.

**Key features**:
- **N**: ~137 countries (varies by year)
- **Time period**: 2021-2023 (3-year rolling averages)
- **Measurement**: Gallup World Poll telephone/face-to-face surveys
- **Sample size**: ~1,000-3,000 respondents per country per year
- **Weighting**: Probability sampling, calibrated to population

### Variables Mapped to PERMA+H

From `01_download_wellbeing_data.py:22-42`:

```python
WHR_COLUMN_MAP = {
    # PERMA+H core
    "positive affect":                    "P_positive_emotion",
    "freedom to make life choices":       "E_engagement_autonomy",
    "social support":                     "R_relationships",
    "ladder score":                       "M_meaning_life_sat",
    "life ladder":                        "M_meaning_life_sat",  # panel edition
    "log gdp per capita":                 "A_accomplishment_gdp",
    "healthy life expectancy at birth":   "H_health",
    "healthy life expectancy":            "H_health",  # mirror variant
    # Reference / robustness columns
    "negative affect":                    "ref_negative_affect",
    "generosity":                         "ref_generosity",
    "perceptions of corruption":          "ref_corruption",
}
```

<Accordion title="Variable Descriptions">
**Cantril Ladder (M - Meaning)**  
Question: "Please imagine a ladder with steps numbered from 0 at the bottom to 10 at the top. The top represents the best possible life for you and the bottom the worst possible life. On which step of the ladder would you say you personally feel you stand at this time?"  
Scale: 0-10 (continuous)  
WHR column: `Ladder score` or `Life Ladder`

**Social Support (R - Relationships)**  
Question: "If you were in trouble, do you have relatives or friends you can count on to help you whenever you need them, or not?"  
Scale: 0 (no) / 1 (yes), aggregated to proportion  
WHR column: `Social support`

**Freedom to Make Life Choices (E - Engagement)**  
Question: "Are you satisfied or dissatisfied with your freedom to choose what you do with your life?"  
Scale: 0 (dissatisfied) / 1 (satisfied), aggregated  
WHR column: `Freedom to make life choices`

**Positive Affect (P - Positive Emotion)**  
Composite: "Did you experience [happiness/enjoyment/smiling or laughing] a lot yesterday?"  
Scale: 0-1 proportion responding "yes" to ≥1 item  
WHR column: `Positive affect` ⚠️ _Often missing in GitHub mirrors_

**Healthy Life Expectancy (H - Health)**  
Source: WHO Global Health Observatory, calibrated to Gallup sample  
Scale: Years (typically 40-80 range)  
WHR column: `Healthy life expectancy` or `Healthy life expectancy at birth`

**Log GDP per Capita (A - Accomplishment)**  
Source: World Bank national accounts, PPP-adjusted  
Scale: Natural log of GDP per capita  
WHR column: `Log GDP per capita`
</Accordion>

## OECD Better Life Index (BLI)

### Description

**Supplemental data** for OECD member countries, providing 11 wellbeing dimensions beyond WHR coverage.

**Key features**:
- **N**: 38 OECD countries
- **Time period**: Latest available year (updated annually)
- **Measurement**: National statistics agencies (official government data)
- **Format**: Long format (Country × Indicator × Inequality dimension)

```python
# From 01_download_wellbeing_data.py:217-258 - pivot logic
def parse_oecd_bli(df: pd.DataFrame) -> pd.DataFrame:
    # Keep "Total" inequality category (average across gender/income groups)
    df = df[df["Inequality"].str.strip().str.lower() == "total"]
    
    # Pivot: one row per country, one column per indicator
    df_wide = (
        df.groupby(["Country", "Indicator"])["Value"]
        .mean()
        .unstack("Indicator")
        .reset_index()
    )
    return df_wide
```

## World Bank Open Data API

### Description

**Supplemental indicators** for health and economic outcomes, filling gaps in WHR coverage.

**Key features**:
- **N**: 200+ countries and territories
- **Time period**: Latest available year (MRV = Most Recent Value API parameter)
- **Measurement**: National statistics from member countries
- **Access**: RESTful API, JSON responses

Coverage: 200+ countries, updated annually

**SL.UEM.TOTL.ZS** - Unemployment, total (% of labor force)

Definition: Level of current health expenditure expressed as % of GDP  
Source: WHO Global Health Expenditure Database  
Coverage: 190+ countries

### How to Access

#### Via REST API (Used by Scripts)

```python
WB_API = "https://api.worldbank.org/v2/country/all/indicator"

def fetch_indicator(indicator_code: str) -> pd.DataFrame:
    url = f"{WB_API}/{indicator_code}?format=json&per_page=300&mrv=1"
    response = requests.get(url, timeout=30)
    payload = response.json()
    
    # Response format: [metadata, data_array]
    records = payload[1] if len(payload) > 1 else []
    
    rows = [
        {
            "country": rec["country"]["value"],
            "iso3": rec["countryiso3code"],
            "value": rec["value"],
            "year": rec.get("date"),
        }
        for rec in records
        if rec.get("value") is not None
    ]
    return pd.DataFrame(rows)
```

## MIDUS 2: Individual-Level Data

### Description

**Individual-level wellbeing data** from the Midlife in the United States longitudinal study (Wave 2, 2004-2006).

<Info>
**Why MIDUS?** This is the **gold standard** for testing the mutualism model at the individual level. Country-level data (WHR) cannot distinguish between:
- True within-person dynamics (mutualism)
- Ecological correlations (between-country differences)

MIDUS provides within-population variation with the same questionnaires across 5,000 people.
</Info>

**Key features**:
- **N**: 4,963 (core sample, Wave 2)
- **Population**: US adults aged 35-86 in 2004
- **Design**: Longitudinal (linkable to Wave 1: 1995-96, Wave 3: 2013-14)
- **Measurement**: Telephone + mail survey (SAQ)
- **Response rate**: 75% from Wave 1 participants

### Ryff Psychological Wellbeing Scale

MIDUS includes **6 composite scores** from Carol Ryff's Psychological Wellbeing Scale:

| Variable | Ryff Dimension | PERMA | Scale | Items |
|----------|---------------|-------|-------|-------|
| `B1SPWBA1` | Autonomy | E | 1-7 | 3 |
| `B1SPWBE1` | Environmental Mastery | A | 1-7 | 3 |
| `B1SPWBG1` | Personal Growth | E | 1-7 | 3 |
| `B1SPWBR1` | Positive Relations | R | 1-7 | 3 |
| `B1SPWBU1` | Purpose in Life | M | 1-7 | 3 |
| `B1SPWBS1` | Self-Acceptance | P | 1-7 | 3 |

**Composite construction**:
- Each subscale = mean of 3 items (sum ÷ 3)
- Original items: 7-point Likert (1 = strongly disagree → 7 = strongly agree)
- Negative items reverse-coded before averaging

### Auxiliary Variables

From `04_individual_level_f_analysis.py:163-164`:

```python
AUX_COLS = ["B1SPOSPA", "B1SNEGAF", "B1SB1", "B1SA11W"]
```

**B1SPOSPA** - Positive Affect  
Composite: Mean of 6 items ("cheerful", "in good spirits", "extremely happy", "calm", "satisfied", "full of life")  
Scale: 1 (none of the time) → 5 (all of the time)  
Correlation with Ryff mean: ~0.55

**B1SNEGAF** - Negative Affect  
Composite: Mean of 6 items ("nervous", "hopeless", "restless", "sad", "worthless", "everything is an effort")  
Scale: 1 (none of the time) → 5 (all of the time)  
Used as **p-factor proxy** (psychopathology)  
Correlation with Ryff mean: ~-0.45

**B1SB1** - Life Satisfaction  
Single item: "Please think about your life as a whole. How satisfied are you with it?"  
Scale: 0 (worst possible) → 10 (best possible)  
Correlation with Ryff Purpose: ~0.60

**B1SA11W** - Self-Rated Health  
Single item: "In general, would you say your physical health is..."  
Original scale: 1 (excellent) → 5 (poor)  
**Scripts reverse-code** to 5 (excellent) → 1 (poor) for consistency  
Correlation with Ryff mean: ~0.35

## Data Harmonization Pipeline

### Merging Strategy

From `01_download_wellbeing_data.py:318-343`:

```python
def merge_sources(whr, bli, wb):
    # 1. Start with WHR as base (most complete PERMA+H coverage)
    merged = whr.copy()
    merged["_key"] = merged["country"].str.lower().str.strip()
    
    # 2. Left join OECD BLI (supplements OECD countries)
    if not bli.empty:
        bli["_key"] = bli["country_oecd"].str.lower().str.strip()
        merged = pd.merge(merged, bli, on="_key", how="left")
    
    # 3. Left join World Bank (fills missing health/economic data)
    if not wb.empty:
        wb["_key"] = wb["country"].str.lower().str.strip()
        merged = pd.merge(merged, wb, on="_key", how="left")
    
    return merged.drop(columns=["_key"])
```

**Key decisions**:
- **WHR as base**: Maximizes coverage (137 countries vs 38 OECD)
- **Fuzzy matching**: Lowercase + strip whitespace (handles "United States" vs "United states")
- **Left join**: Never lose WHR countries (supplemental sources only add columns)
- **No imputation**: Missing values left as NaN (complete-case analysis downstream)

### Column Naming Convention

| Prefix | Source | Example |
|--------|--------|----------|
| `P_`, `E_`, `R_`, `M_`, `A_`, `H_` | WHR (PERMA+H primary) | `M_meaning_life_sat` |
| `oecd_` | OECD BLI | `oecd_life_satisfaction` |
| `wb_` | World Bank | `wb_life_expectancy` |
| `ref_` | Reference (robustness checks) | `ref_negative_affect` |

## Data Quality Notes

### Known Issues

<Warning>
**Positive Affect Missing**: WHR GitHub mirrors often exclude the "Positive affect" column. Check missingness report:
```python
print(f"P_positive_emotion: {df['P_positive_emotion'].isna().sum()} / {len(df)} missing")
```
If >50% missing, exclude P from analysis or impute from life satisfaction.
</Warning>

**Country Name Variations**:
- WHR: "United States", OECD: "United States of America", WB: "United States"
- Scripts use fuzzy matching (lowercase + strip) but edge cases exist
- Check `_report_missingness()` output for unexpected gaps

**Temporal Mismatch**:
- WHR: 3-year average (2021-2023)
- OECD: Latest year varies by indicator (2020-2023)
- World Bank: MRV (could be 2015 for some countries/indicators)
- Not a problem for cross-sectional analysis, but be cautious with causal interpretation

**Scale Differences**:
- WHR Life Satisfaction: 0-10 continuous
- MIDUS Life Satisfaction: 0-10 continuous (comparable!)
- WHR Healthy Life Expectancy: 40-80 years
- MIDUS Self-Rated Health: 1-5 Likert (not comparable without scaling)

### Recommended Preprocessing

From `02_analyze_wellbeing_f.py:118-143`:

```python
def load_data():
    df = pd.read_csv("wellbeing_data/wellbeing_merged.csv")
    
    # 1. Select PERMA+H columns present in data
    present = [c for c in PERMA_COLS if c in df.columns]
    
    # 2. Complete-case analysis (drop rows with ANY missing)
    df_clean = df[["country"] + present].dropna(subset=present)
    print(f"Complete PERMA+H cases: {len(df_clean)} / {len(df)}")
    
    # 3. Standardize (z-score) - CRITICAL for PCA
    scaler = StandardScaler()
    X_scaled = scaler.fit_transform(df_clean[present].values)
    
    return df_clean, X_scaled, present
```

**Why standardize?**
- Dimensions have different scales (0-1 for social support, 0-10 for life satisfaction)
- PCA is scale-sensitive (large-scale variables dominate)
- f-factor loadings interpretable as correlations after standardization
