### Step 1: Download and Merge Data Sources

Run the data download script first:

```bash

```

- Downloads World Happiness Report 2024 data from GitHub

- Maps columns to PERMA+H schema
- Merges all sources on country names
- Outputs `wellbeing_data/wellbeing_merged.csv`

- Generates 7 publication-quality figures
- Saves scored dataset with f values

## Common Issues and Solutions

### "File not found: wellbeing_data/wellbeing_merged.csv"

**Solution:** Run `01_download_wellbeing_data.py` first. Scripts must be run in sequence.

### "Connection timeout" during data download

**Solution:** Check your internet connection. If World Happiness Report GitHub mirror is down, you can manually download from the [official WHR site](https://worldhappiness.report) and place it in `wellbeing_data/WHR2024_raw.csv`.

### Low omega_h (less than 0.60) in individual-level data

**Cause:** Individual-level data naturally has more noise than country aggregates. Country-level analysis inflates correlations through ecological aggregation (Simpson's paradox).

**Expected:** omega_h ≈ 0.50-0.65 at individual level vs. 0.70-0.85 at country level.

### "semopy not installed" warning

**Solution:** This is optional. If you want bifactor CFA results:
```bash
pip install semopy
```

PCA results (PC1 as f proxy) are already provided and are the primary analysis.

### Missing PERMA+H columns after data download

**Cause:** Column names in WHR data may change between editions.

**Solution:** Check `WHR_COLUMN_MAP` in `01_download_wellbeing_data.py`. The script includes multiple aliases for each column to handle naming variations.

### "Not all correlations positive" warning

**Cause:** May indicate data quality issues or incorrect variable mapping.

**Check:**
1. Are all variables scaled in the same direction? (higher = better wellbeing)
2. Did script 01 correctly map columns?
3. Are there extreme outliers?

Run this diagnostic:
```python
import pandas as pd
df = pd.read_csv('wellbeing_data/wellbeing_merged.csv')
print(df[['P_positive_emotion', 'H_health']].describe())
```

## Batch Execution

To run the complete pipeline in one command:

```bash
python 01_download_wellbeing_data.py && \
python 02_analyze_wellbeing_f.py && \
python 03_models_full_f.py
```

Or create a shell script:

```bash
#!/bin/bash
set -e  # Exit on error

echo "Step 1: Downloading data..."
python 01_download_wellbeing_data.py

echo "Step 2: Running f-factor analysis..."
python 02_analyze_wellbeing_f.py

echo "Step 3: Full model implementation..."
python 03_models_full_f.py

echo "✓ Analysis complete. Results in wellbeing_figures/"
```

Save as `run_analysis.sh` and execute:
```bash
chmod +x run_analysis.sh
./run_analysis.sh
```
