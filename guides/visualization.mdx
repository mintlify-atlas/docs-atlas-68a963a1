### 02_scree_plot.png

<Accordion title="Eigenvalue Decomposition (2 panels)">

**Left panel: Scree plot**
- X-axis: Principal component number (1-6)
- Y-axis: Eigenvalue (variance explained)
- Blue bars: PC1 is highlighted
- Orange dashed line: Kaiser criterion (λ = 1)

**What it shows:**
```
PC1 eigenvalue = 4.097  (explains 68.3%)
PC2 eigenvalue = 0.782  (explains 13.0%)
PC3 eigenvalue = 0.521  (explains 8.7%)
...
```

**The "elbow":**
- Sharp drop after PC1 → Dominant general factor
- Gradual decline after PC2 → Domain-specific variance

**Kaiser rule:** Keep components with λ > 1
- Here: Only PC1 qualifies → Unidimensional f is justified

**Right panel: Cumulative variance**
- X-axis: Number of components
- Y-axis: Cumulative % variance explained
- Step function: Each step adds one component
- Gray dashed line: 80% threshold

**Example:**
```
PC1:           68.3%
PC1+PC2:       81.3%  ← Crosses 80% threshold
PC1+PC2+PC3:   90.0%
```

**Interpretation:**
- 1 component needed for 68% variance
- 2 components needed for 80% variance (common target)
- But PC2 eigenvalue < 1 → Not a "real" factor, just residual

**Code reference:**
```python
# From plot_scree() in 02_analyze_wellbeing_f.py:323

ax1.axhline(y=1.0, color="#E8622A", ls="--", lw=1.4,
            label="Kaiser criterion (λ = 1)")
```

**Saved to:** `wellbeing_figures/02_scree_plot.png`

<Accordion title="PC1 Loadings with Bootstrap CIs">

**Example:**

```
R: Relationships             0.423  [0.368, 0.475]

P: Positive Emotion          0.398  [0.312, 0.472]
A: Accomplishment (GDP)      0.387  [0.318, 0.448]  ← Lowest
```

**How to read error bars:**
- **Narrow CI** → Stable loading (low sampling variability)
- **Wide CI** → Unstable loading (high uncertainty)
- **CIs overlap** → Loadings not significantly different
- **CIs don't overlap** → Significantly different loadings

**Interpretation:**
- All loadings in 0.38-0.43 range → Roughly equal weighting
- This is the signature of a **broad general factor**
- No single dimension dominates f

### 05_f_distribution.png

<Accordion title="Histogram and KDE of f-Scores">

**What it shows:**
- Blue histogram: Empirical distribution of country f-scores
- Orange line: Kernel density estimate (smoothed)
- Black dashed line: N(0,1) reference (theoretical normal)
- Vertical gray line: Mean (should be at 0)

**Statistical test annotation:**
```
Shapiro-Wilk: W=0.984, p=0.089
```

**Interpretation:**
- W close to 1 → Distribution is approximately normal
- p > 0.05 → Cannot reject normality (good)
- If p < 0.05 → Distribution is significantly non-normal (skewness/outliers)

**Code reference:**
```python

        label="N(0,1) reference", alpha=0.6)
```

**What to look for:**
- **Symmetric bell curve** → Normal distribution (expected)
- **Left skew** → Few very low-f countries
- **Right skew** → Few very high-f countries
- **Bimodal** → Two clusters (e.g., developed vs. developing)

### 07_coupling_proxy.png

<Accordion title="Partial Correlation Matrix">

**What it shows:**
- Heatmap of partial correlations (controlling for all other dimensions)
- Color scale: Blue (negative) ← White (zero) → Red (positive)
- Symmetry: Both triangles shown (unlike correlation matrix)

**Partial correlation = direct relationship** between two dimensions after removing indirect effects through other dimensions.

**Code reference:**
```python
# From plot_coupling_proxy() in 02_analyze_wellbeing_f.py:577
# Precision matrix = inverse of correlation matrix
C    = np.corrcoef(X.T)
P    = inv(C)
# Partial correlations: r_ij|rest = -P_ij / sqrt(P_ii * P_jj)
D    = np.sqrt(np.diag(P))
pc   = -P / np.outer(D, D)
```

**Interpretation:**
- **Zero-order r = 0.80, partial r = 0.10** → Relationship is mostly indirect
- **Zero-order r = 0.60, partial r = 0.55** → Relationship is direct (independent)
- **Partial r ≈ zero** → No direct coupling (relationship mediated by others)

**Mutualism model connection:**
- Partial correlations ≈ mutualistic coupling matrix M
- Strong partial r → Strong direct mutual facilitation
- Example: If Relationships directly support Health (not through Meaning)

**What to look for:**
- **Positive partial r** → Direct mutual reinforcement
- **Negative partial r** → Suppression effect (rare in wellbeing data)
- **Near-zero partial r** → Indirect relationship (explained by other dimensions)

**Saved to:** `wellbeing_figures/07_coupling_proxy.png`

</Accordion>

### 10_lyapunov_analysis.png

<Accordion title="Model 3: Three-Panel Stability Analysis">

**(a) Jacobian Eigenvalues**
- **Scatter plot:** Real vs imaginary parts of eigenvalues
- **Red dashed line:** Re(λ) = 0 (stability boundary)
- **Annotation:** Dominant eigenvalue λ₁ (closest to zero)

**Interpretation:**
- **All points left of red line (Re < 0)** → Stable equilibrium
- **Any point right of red line (Re > 0)** → Unstable (runaway dynamics)
- **Close to boundary** → Slow recovery (low resilience)

```python
# From plot_lyapunov_analysis() in 03_models_full_f.py:810

           edgecolors="black", zorder=5)

           label="Stability boundary")
```

**(b) Recovery from Perturbation**
- **Time series:** % deviation from equilibrium after shock
- **Shock:** 30% reduction to first domain at t=0
- **All lines converge to 0** → System returns to equilibrium
- **Convergence speed ≈ 1/τ** (return time)

**Example:**
```
τ = 12.4
Shock recovery: 63% after 12.4 time units, 99% after ~62 units
```

**(c) Basin of Attraction (2D projection)**
- **Ellipse:** Boundary of basin (region where system returns to equilibrium)
- **Star at center:** Equilibrium point x*
- **Ellipse size:** Proportional to resilience R(x*)

**Interpretation:**
- **Large ellipse** → Deep basin → High resilience
- **Small ellipse** → Shallow basin → Low resilience
- Perturbations inside ellipse → Return to x*
- Perturbations outside ellipse → May diverge or reach different equilibrium

## Figure Export Settings

All figures use these settings (defined in matplotlib rcParams):

```python
matplotlib.rcParams.update({
    "figure.dpi":    120,      # Screen display
    "savefig.dpi":   150,      # Saved file (suitable for papers)
    "savefig.bbox":  "tight",  # Remove excess whitespace
    "savefig.format": "png",   # Default format
})
```

**For publication:**
Change DPI to 300 and export as PDF or SVG:
```python
fig.savefig("figure.pdf", dpi=300, bbox_inches="tight")
fig.savefig("figure.svg", bbox_inches="tight")  # Vector format
```
